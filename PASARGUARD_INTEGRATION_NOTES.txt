PasarGuard integration added by ChatGPT
=====================================

What changed:
- Added a new server type: pasarguard / پاسارگارد
- Added PasarGuard API login via: POST /api/admin/token
- Added PasarGuard user creation via: POST /api/user
- Bot creates one PasarGuard user per order and sends the PasarGuard subscription_url as the subscription link.
- For PasarGuard, the old inbound_id field is used as PasarGuard group_id when creating users.

How to add your PasarGuard panel in the bot:
1) Bot admin panel -> servers -> ثبت پنل پاسارگارد
2) Panel URL: your PasarGuard panel URL, e.g. https://domain:8000
3) Username/password: PasarGuard admin username/password
4) If you use the normal x-ui server add flow, set server type to پاسارگارد.

Important for plans:
- In PasarGuard, users must have a Group that contains the Host/Inbound tags.
- In the bot plan's inbound_id, put the PasarGuard group ID.
- Example: if your PasarGuard group ID is 1, set inbound_id = 1 for that plan.

Optional advanced plan settings:
- You can put JSON in server_plans.custom_sni to override PasarGuard creation settings:
{
  "group_ids": [1,2],
  "proxy_settings": {},
  "note": "created by bot"
}

Test flow:
1) Buy a small test plan.
2) Check PasarGuard dashboard -> Users: user should be created with remark like SERVER-USERID-RANDOM.
3) Bot should send subscription link from PasarGuard.

Limitations:
- This version focuses on creation + subscription delivery.
- Basic delete/disable/edit routes were added, but heavy management flows should be tested on one test user first.
